Sequence Mirrors

The Sequence system mirrors YAML data into SQLite databases under User/Data/ so dashboards and agents can query fast without reparsing files.

Databases

  • chronos_core.db - canonical mirror of items, relations, completions, and schedules.
  • chronos_matrix.db - analytics cache for cockpit Matrix panels.
  • chronos_events.db - listener logs and command history.
  • chronos_memory.db - activity facts and status snapshots.
  • chronos_trends.db + trends.md - completion and variance digest.

Commands

  • sequence status - list mirrors from User/Data/databases.yml.
  • sequence sync <targets> - rebuild one or more datasets (matrix, core, events, memory, trends).
  • sequence trends - rebuild memory/trends and rewrite User/Data/trends.md.

Automation

The Listener runs sequence sync memory trends shortly after midnight. State lives in User/Data/sequence_automation.yml.
